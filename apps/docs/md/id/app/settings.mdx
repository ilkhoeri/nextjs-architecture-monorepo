# Halaman Pengaturan

---

Halaman dengan awalan `/settings` diperuntukkan secara eksklusif untuk **_current user_**—yakni pengguna yang saat ini sedang login. Setiap parameter (`params`) di URL digunakan untuk meninjau, memperbarui, atau menghapus data berdasarkan akun tersebut. Untuk saat ini, hanya terdapat satu parameter aktif, yaitu berdasarkan `username`, yang berfungsi untuk memperbarui data pengguna sesuai kebutuhan UX dan spesifikasi tim YouApp.

## &#91;username&#93;

Parameter ini diambil dari nilai `username` masing-masing pengguna. Oleh karena itu, di dalam database, field `username` pada `model User` bersifat **unik** (unique constraint). Halaman ini memungkinkan pengguna untuk melihat dan memperbarui data secara terstruktur melalui beberapa tampilan **kartu**, antara lain:

### Kartu Pertama

---

Pada Kartu pertama akan menampilkan beberapa informasi seperti:

- **Foto profil**
- **Nama**
- **Gender**
- **Horoscope (zodiak barat)**
- **Zodiac (shio Tionghoa)**

Untuk **Badge Horoscope** dan **Badge Zodiac**, pengguna dapat mengklik elemen tersebut untuk menampilkan informasi detail sesuai dengan tanggal dan tahun lahir yang telah dimasukkan.

#### Data Referensi:

- **Zodiac (Shio)** diperoleh melalui metode scraping yang aman dari situs [yourchineseastrology.com](https://www.yourchineseastrology.com), menggunakan [Cheerio](https://cheerio.js.org/docs/intro).
- **Horoscope (Zodiak Barat)** diperoleh melalui public API dari [API Ninjas](https://api.api-ninjas.com/v1/horoscope) menggunakan HTTP fetch (membutuhkan `NINJAS_PUBLIC_KEY`).

#### UX Enhancement:

Ketika lebar layar melebihi 768px (asumsi query _notebook_) dan pengguna belum memiliki foto profil, maka pengguna dapat melakukan **drag & drop** langsung ke dalam kartu untuk mengunggah gambar baru.

### Kartu Kedua

---

Kartu kedua digunakan untuk memperbarui informasi pribadi pengguna secara serentak:

- **Foto Profil** <br />
  Opsional. File media (`.jpg`, `.gif`, `.mp4`) akan disimpan di [Cloudinary](https://cloudinary.com/), dan database menyimpannya sebagai URL. <br />
  Jika pengguna mendaftar menggunakan layanan third party (mis. Google), URL foto akan diambil dari provider terkait.

- **Nama** <br />
  Pengguna dapat memperbarui nama yang digunakan dengan ketentuan minimal dua karakter. Jika terdiri dari lebih dari satu kata, nama akan di-_mapping_ di field `firstName` dan `lastName` dalam database. <br />
  Contoh: `Alexander Immanuel Johndoe` → `firstName = Alexander`, `lastName = Immanuel Johndoe`.

- **Gender** <br />
  Pengguna dapat menyesuaikan gender sesuai pilihan yang ada.
  Pilihan gender yang tersedia adalah `Male` dan `Female`.

- **Tanggal Lahir** <br />
  Digunakan untuk:
  - Validasi internal.
  - Kalkulasi `Horoscope` dan `Zodiac`. <br />
    _Catatan: Jika `birthDate` bernilai `null` atau `undefined`, maka keduanya tidak akan ditampilkan._

- **Zodiac (Shio Tionghoa)** <br />
  - Dihitung berdasarkan tanggal-bulan-tahun lahir.
  - Mendukung:
    - Kalkulasi siklus shio sampai tahun **2043-02-10**.
    - Kategori `animals` dan `elements`.
    - Informasi deskriptif.
    - Tampilan ikon yang sesuai.

- **Horoscope (Zodiak Barat)** <br />
  - Dihitung berdasarkan tanggal-bulan lahir.
  - Info singkat, dan tampilan yang informatif.
  - Tampilan ikon yang sesuai.

- **Tinggi Badan** <br />
  Berupa angka, dinyatakan dalam satuan **sentimeter (cm)**. _Asian users friendly._

- **Berat Badan** <br />
  Berupa angka, dinyatakan dalam satuan **kilogram (kg)**.

### Kartu Ketiga

---

Kartu ini menampilkan daftar **Interest** pengguna.
Ketika pengguna mengklik ikon `✏️` `(pencil/edit)`, kartu akan melebar dan menampilkan input interest. Perubahan dilakukan dengan animasi halus (efek stretch/expand).
Pengguna dapat memilih dari opsi interest yang sesuai atau menambah, mengubah, atau menghapus interest mereka.

### Kartu Keempat

---

Digunakan untuk memperbarui **Password** pengguna.
Form password membutuhkan password pengguna saat ini untuk validasi akun. Terdapat `Password Requirement` untuk memandu pengguna dalam pembuatan password baru sesuai ketentuan.

### Kartu Kelima

---

Pengguna dapat menambahkan, mengubah, atau menghapus **tautan media sosial** miliknya, seperti Instagram, LinkedIn, Twitter, GitHub, dan lainnya.

### Kartu Keenam

---

Kartu terakhir menampilkan **perangkat dan sesi login pengguna**.

- Lokasi perangkat ditentukan berdasarkan izin pengguna.
- Fitur ini dapat dikembangkan lebih lanjut untuk menyimpan data session ke database (`ping`), dan memungkinkan integrasi dengan tools seperti [`pusher-js`](https://pusher.com/) agar pengguna dapat melihat lokasi atau status sesi aktif secara real-time.
